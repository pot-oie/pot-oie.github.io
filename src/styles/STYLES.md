# 项目设计系统与样式规范 (Design System & Specs)

本项目基于 Astro + Tailwind CSS 开发，核心视觉风格为**“现代水墨 (Modern Ink-wash)”**，注重极简排版、呼吸感以及中文字体之美。请在生成 HTML/CSS/Tailwind 代码时严格遵循以下规范。

## 1. 核心色彩系统 (Color System)

系统围绕“墨色阶梯”与“朱红印章”展开，支持深浅双色模式(`.dark`)。

- **墨色 (Ink) - 主导色**:
  - `--color-ink-900` (#0f0f0f): 焦墨 - 用于极深色背景或强调文本。
  - `--color-ink-700` (#2a2a2a): 浓墨 - 用于正文核心阅读区。
  - `--color-ink-500` (#555555): 重墨 - 用于次要文本、注释。
  - `--color-ink-300` (#a3a3a3): 淡墨 - 用于线条、边框、禁用状态。
  - `--color-ink-100` (#f5f5f5): 纸色 - 用于亮色背景。
- **强调色 (Accent)**:
  - `--color-vermilion` (#b91c1c): 朱红 - 核心交互色（链接悬停、选中状态、水墨风重点边框等），具有印章质感。
- **基础映射 (Light / Dark)**:
  - 背景: `#f2f2f2` / `#18181b` (Zinc-900)
  - 卡片: `#ffffff` / `#27272a` (Zinc-800)
  - 文字: 深墨色 / `#e4e4e7` (Zinc-200)

## 2. 字体与排版 (Typography & Prose)

全局使用 `.prose-ink` 类包裹 Markdown 渲染内容，限制最大宽度 `65ch`，行高 `2`。

- **字体栈**:
  - 无衬线 (Sans) `Inter` - 用于基础 UI 与正文。
  - 有衬线 (Serif) `Noto Serif SC` - 用于**所有标题 (h1-h6)**、加粗强调、表格表头、公式及引用样式。
- **排版细节**:
  - **链接 (Links)**: 默认下划线为淡墨色，Hover 时文字和下划线变为**朱红色**。
  - **选中色 (Selection)**: 亮色模式为焦墨底白字，暗色模式为**朱红底白字**。
  - **列表 (Lists)**: 无序列表使用自定义的朱红旋转方块（菱形）作为 marker；有序列表使用衬线斜体数字。
  - **引用 (Blockquote)**: 亮色为透明底+深墨左边框，暗色为微白底+朱红左边框。
  - **分割线 (HR)**: 两侧透明渐变，正中间带有衬线符 `§`。

## 3. 全局特效与材质 (Global Effects)

- **白噪点材质**: `body::before` 使用了一层 `opacity: 0.05` 的 SVG `fractalNoise`，不要覆盖掉 body 的相对定位。
- **阴影**: 统一使用变量 `--shadow-ink` (类似 `0 10px 40px -10px rgba(0,0,0,0.15)`) 模拟墨水晕染的柔和边缘。

## 4. 代码块组件规范 (Code Blocks)

代码块使用统一外层容器 `.code-block-wrapper`，内置**两套UI风格**，不要混用：

1.  **macOS 风格 (默认)**:
    - 视觉: 圆角边框，带有柔和阴影。
    - 头部: 带红黄绿三个“红绿灯”控制点（径向渐变实现），右上角有大写的语言标识。
    - 交互: 绝对定位的浮动按钮（复制、折叠、主题切换），悬停变朱红。自定义 macOS 风格滚动条。
2.  **水墨风格 (附加类 `.style-ink`)**:
    - 视觉: 去除边框与圆角，左侧加粗 `3px` 朱红边框。背景使用从左至右的线性渐变（淡墨色至透明）。
    - 水印: 语言标识化为巨大的浅色透明水印字体，放置在代码块右侧底层。
    - 行号: 朱红色并降低透明度。

## 5. KaTeX 公式样式 (Math)

- **行内公式**: 增加微小的呼吸边距与极淡的背景色，移动端允许防溢出横向滚动（隐藏滚动条）。
- **块级公式**: 带有代码块相似的背景渐变和淡墨色左边框。Hover 时，左边框变为朱红色，背景泛起微弱的朱红渐变。
- **滚动提示**: 配合超长内容，包含一个 `.scroll-hint-tail` 元素，带有 `tailBreathe` (左右呼吸移动) 动画的斜体衬线提示文本。

## 6. 移动端适配法则 (Mobile Responsive)

在 `< 640px` 时：

- 所有正文、列表、引用字号缩小。
- **代码块越界展开**: 代码块及深层列表内的代码容器使用负 `margin` (`-1rem` 或 `-2em`) 撑满屏幕宽度，打破常规排版边界。行号向左偏移并压缩间距。
