> 如果说 Stable Diffusion 是把扩散过程搬到潜空间，那么 DiT 更进一步：把去噪器本身从 U-Net 换成 Transformer。

[前面](/blog/diffusion-03-latent-diffusion/)讲到，Latent Diffusion 的核心是先用 VAE 把像素图压缩成 latent，再让 U-Net 在 latent space 里完成去噪。

这套方案很成功，但它仍然保留了一个 CNN 时代的核心假设：图像生成的主干网络最好是 U-Net。

DiT（Diffusion Transformer）要问的问题正好相反：

> 既然 Transformer 已经统一了文本、视觉和多模态任务，扩散模型里的 denoiser 能不能也直接换成 Transformer？

## 从 U-Net 到 Transformer

U-Net 很适合图像去噪，因为它天然保留二维空间结构：

- 下采样负责获得大范围语义。
- 上采样负责恢复空间细节。
- Skip Connection 负责把浅层纹理接回去。

但 U-Net 也有一个问题：它的结构强依赖 CNN 归纳偏置。局部卷积、金字塔式分辨率变化、跨层拼接，这些设计都很适合图像，但也让模型扩展时更像在堆一套视觉专用工程。

Transformer 的优势则是另一种：

- 所有 token 之间可以直接建模全局关系。
- 深度、宽度、token 数量都可以用比较统一的方式扩展。
- 语言模型已经证明了 Transformer scaling 的威力。

所以 DiT 的核心不是改扩散过程，而是改 denoiser：

$$
\epsilon_\theta(z_t, t) : \text{U-Net} \to \text{Transformer}
$$

## Patchify Latent

DiT 并不是直接把原始像素扔进 Transformer。

它通常接在 Latent Diffusion 后面：先用 VAE Encoder 把图像压缩成 latent，再把 latent 切成一块块 patch。

例如一个 $32 \times 32 \times C$ 的 latent feature map，可以按 $2 \times 2$ 的 patch 切开，变成一串 token：

$$
z_t \in \mathbb{R}^{H \times W \times C}
\to
\text{tokens} \in \mathbb{R}^{N \times D}
$$

其中 $N$ 是 patch token 数量，$D$ 是每个 token 的 hidden dimension。

这一步和 ViT 很像：图像不再被看成二维网格，而是被看成 token sequence。

## 时间与条件怎么注入

扩散模型的 denoiser 不能只看 $z_t$，还必须知道两个信息：

1. 当前处于第几步去噪：时间步 $t$。
2. 生成条件是什么：类别标签、文本 prompt 或其他条件。

在 U-Net 里，时间通常通过 Time Embedding 注入到 ResBlock，文本通过 Cross-Attention 注入。

DiT 的做法更 Transformer 化：把时间和条件编码成 conditioning vector，再用它去调制 Transformer block。

原论文里一个重要设计是 **adaLN-Zero**。可以粗略理解为：

- 先对 Transformer block 做 LayerNorm。
- 再根据时间和条件生成一组缩放、平移和门控参数。
- 用这些参数调节 block 的输出强度。

这样模型不是简单“把条件拼进去”，而是在每一层里动态改变特征流动方式。

## DiT Block

一个 DiT block 可以粗略理解为标准 Transformer block 的扩散版本：

1. Multi-Head Self-Attention：让所有 latent patch 彼此通信。
2. MLP：对每个 token 做非线性变换。
3. Conditioning：用时间步和类别/文本条件调制 block。

如果和 U-Net 对比：

- U-Net 依赖卷积和多尺度结构来传播图像信息。
- DiT 依赖 Self-Attention 直接建模 patch 之间的全局关系。

这也是 DiT 最值得放在 Transformer 之后学的原因：它不是“扩散模型里多加了注意力”，而是整个 denoiser 都换成了 Transformer。

换句话说，DiT 的成立依赖于前面 Transformer 已经证明过的一件事：**Self-Attention 可以作为通用的 Token Mixer**。

在语言模型里，Token 是文本片段；在 ViT 里，Token 是图像 Patch；到了 DiT 这里，Token 则变成了被 VAE 压缩后的 Latent Patch。Attention 机制本身不关心这些 Token 原本来自文字、像素还是潜空间，它只关心一件事：

> 当前 Token 应该从其他 Token 中读取哪些信息？

所以 DiT 并不是凭空把 NLP 架构硬塞进图像生成，而是沿着同一条抽象路线继续走：先把输入变成 Token Sequence，再用 Self-Attention 做全局信息混合，最后把结果还原回原来的空间结构。

## 为什么 DiT 重要

DiT 论文的重点不只是“Transformer 也能做扩散”，而是它展示了一个更重要的趋势：**扩散模型同样可以吃 scaling law。**

论文用 Gflops 衡量前向计算量，并发现：

- 增加 Transformer 深度。
- 增加 hidden dimension。
- 减小 patch size，增加 token 数量。

这些都会提升计算量，而更大的 DiT 通常能得到更好的生成质量。

这和语言模型里的经验很像：当模型结构足够统一，性能提升就越来越依赖规模、数据和算力。

## 和 Stable Diffusion 的关系

Stable Diffusion 的主线可以概括为：

$$
\text{VAE latent} + \text{U-Net denoiser} + \text{Text conditioning}
$$

DiT 则可以概括为：

$$
\text{VAE latent} + \text{Transformer denoiser} + \text{Conditioning}
$$

所以 DiT 不是推翻 Latent Diffusion，而是在 Latent Diffusion 的框架里替换主干网络。

更准确地说：

- DDPM 解决了“扩散模型怎么训练”。
- DDIM / DPM-Solver 解决了“怎么更快采样”。
- Latent Diffusion 解决了“怎么在更便宜的空间里生成”。
- DiT 解决了“denoiser 能不能更像现代大模型一样扩展”。
