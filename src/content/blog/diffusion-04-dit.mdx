DiT（Diffusion Transformer）提问：

> 既然 Transformer 已经统一了文本、视觉和多模态任务，扩散模型里的 denoiser 能不能也直接换成 Transformer？

## 从 U-Net 到 Transformer

U-Net 很适合图像去噪，因为它天然保留二维空间结构：

- **下采样（Downsampling）**：负责提取全局的大范围语义。
- **上采样（Upsampling）**：负责逐步恢复空间分辨率与细节。
- **跳跃连接（Skip Connection）**：负责将浅层未破坏的局部纹理直接传递给深层，防止细节丢失。

但 U-Net 也面临扩展性问题。它的网络架构强依赖于 CNN 的归纳偏置（Inductive Bias）。局部感受野、金字塔式的分辨率变化、跨层特征拼接，这些设计固然契合图像特性，但也导致模型在扩大规模（Scaling up）时缺乏统一的标准，往往需要大量针对视觉的定制化工程调优。

Transformer 的优势恰好在另一面：

- 通过全局的自注意力机制（Self-Attention），所有 Token 之间可以直接建模远距离依赖，无需堆叠多层卷积来扩大感受野。
- 网络深度的增加、宽度的拓展、Token 数量的放缩，都可以遵循一套高度统一的规则。
- 最重要的是，大语言模型（LLM）已经充分验证了 Transformer 架构在应对海量数据和参数规模时的 Scaling Law。

因此，DiT 的核心创新并不在于修改扩散的加噪/去噪数学过程，而是直接在算子层面进行替换：

$$\epsilon_\theta(z_t, t) : \text{U-Net} \to \text{Transformer}$$

## 潜空间切片（Patchify Latent）

DiT 并不会将原始像素直接输入 Transformer（计算复杂度会呈平方级爆炸）。它的标准做法是接在 Latent Diffusion 的前端之后：先利用预训练的 VAE Encoder 将高分辨率图像压缩到潜空间，随后将该 Latent Feature Map 切分成一个个连续的 Patch。

假设经过 VAE 压缩后的潜变量尺寸为 $H \times W \times C$，我们设定切片大小为 $P \times P$。那么，这个三维张量会被展平并映射为一串长度为 $N$ 的 Token 序列：

$$z_t \in \mathbb{R}^{H \times W \times C} \to \text{tokens} \in \mathbb{R}^{N \times D}$$

其中：

- $N = \frac{H \times W}{P^2}$：表示切分后的 Patch 总数，也就是序列的 Token 长度。
- $D$：表示每个 Token 经过线性投影（Linear Projection）后的隐藏层维度。

这一步在数学形式和工程实现上与 ViT（Vision Transformer）完全一致。图像不再被视为具有绝对 $X-Y$ 坐标的二维像素网格，而是被抽象为一组离散的 Token Sequence。

## 条件注入与 adaLN-Zero

在标准的无条件生成之外，扩散模型的去噪网络必须实时接收两种核心引导信息：

1. **时间步 $t$**：指示当前处于去噪过程的哪个阶段（噪声水平）。
2. **生成条件 $c$**：如类别标签（Class Label）或文本提示（Text Prompt）。

在 U-Net 架构中，时间信息通常通过 Time Embedding 加到 ResNet 块中，文本条件则通过 Cross-Attention 交叉映射。而在 DiT 中，这些额外信息的注入方式更加“Transformer 化”——通过自适应层归一化（Adaptive Layer Normalization, adaLN）来实现对网络块的动态调制。

DiT 论文提出了一种名为 **adaLN-Zero** 的特殊设计。我们将时间和条件向量映射为每一层需要的缩放（Scale, $\gamma$）、平移（Shift, $\beta$）和门控（Gate, $\alpha$）参数：

$$\text{adaLN}(x, c, t) = \gamma(c, t) \cdot \text{LayerNorm}(x) + \beta(c, t)$$

更关键的是“Zero”的含义：在模型初始化时，控制多层感知机（MLP）和多头注意力（MSA）残差分支输出的门控参数 $\alpha$ 会被严格初始化为 0。这意味着在训练的第一步，每个 DiT Block 的行为等价于恒等映射（Identity Function）。这种策略极大地稳定了深层 Transformer 的早期训练梯度。

## DiT Block 结构

剥离掉条件注入模块后，一个 DiT Block 的核心几乎就是最标准的 Transformer Block：

1. **多头自注意力（Multi-Head Self-Attention）**：负责让所有 Latent Patch 之间相互通信，进行全局的特征重组。
2. **前馈网络（MLP）**：对每个 Token 的特征维度进行独立的非线性升维与降维。
3. **条件调制（Conditioning）**：利用前文提到的 adaLN-Zero 机制，根据时间步和类别/文本条件来缩放特征流动。

如果拿它和传统的 U-Net 做本质区别的对比：

- U-Net 依赖**卷积核的物理滑动**和**多尺度上下采样**来强行传播二维图像的局部语义。
- DiT 则依赖**自注意力矩阵的点积运算**，直接建立不同 Patch 之间的全局关联权重。

DiT 并不是“在扩散模型里加了一点注意力层”，而是彻底移除了卷积，证明了 **Self-Attention 完全可以作为扩散去噪过程的通用 Token Mixer**。无论是处理一段自然语言文本，还是处理被 VAE 压缩过的一组图像潜变量特征，注意力机制只关心一件事：当前 Token 应该从序列的其他 Token 中读取多少信息。它彻底统一了不同模态在底层特征交互时的数学形式。

## 规模效应（Scaling Law）

DiT 论文最重要的结论，不是论证了 Transformer 能做扩散，而是证明了 **扩散模型同样遵循可预测的 Scaling Law**。

通过引入计算复杂度指标 Gflops（每秒十亿次浮点运算），论文揭示了决定模型生成质量（如 FID 分数）的三个核心变量：

1. **增加 Transformer 深度**（层数）。
2. **增加隐藏层维度 $D$**（网络宽度）。
3. **减小 Patch Size $P$**（直接导致 Token 数量 $N$ 呈平方级增加）。

实验表明，计算量（Gflops）的增加与生成质量的提升呈现出高度稳定的正相关。这与大语言模型的发展轨迹如出一辙：当基础架构足够纯粹且统一时，人工干预的“Trick”就不再重要，模型能力的上限将直接由算力、数据和参数规模来决定。

## 与 Stable Diffusion 演进的关系

要理清这几代核心模型的定位，可以将生成范式拆解为不同的模块组合。

Stable Diffusion 的主线公式为：

$$\text{VAE Latent} + \textbf{U-Net Denoiser} + \text{Text Conditioning}$$

而 DiT 的主线公式为：

$$\text{VAE Latent} + \textbf{Transformer Denoiser} + \text{Conditioning}$$

DiT 并不是对 Latent Diffusion（潜在扩散）框架的否定，而是对其内部计算引擎的升级。纵观整个技术路线图：

- **DDPM** 确立了底层数学基础：解决了“扩散过程如何定义与训练”。
- **DDIM / DPM-Solver** 优化了解码过程：解决了“如何用更少的步数完成采样”。
- **Latent Diffusion** 优化了空间维度：解决了“如何在低维空间高效生成高分辨率图像”。
- **DiT** 优化了网络架构：解决了“去噪器如何突破 CNN 瓶颈，像大语言模型一样无限吃进算力并兑现为能力”。
